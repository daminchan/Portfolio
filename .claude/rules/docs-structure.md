# ドキュメント構造

## 情報タイプの分類（DITA 準拠）

| タイプ | 内容 | 配置場所 |
|--------|------|----------|
| Concept（ルール・判断基準） | 何をすべきか、なぜか | `rules/` |
| Task（手順・チェックリスト） | どうやるか | `skills/` |
| Reference（設計データ・仕様） | 参照情報 | `docs/` |

## rules/ の作成ルール

- **MUST**: 1ファイル1トピック
- **MUST**: 箇条書き形式（MUST/SHOULD/NEVER で強度を明示）
- **MUST**: 具体的に記述する（「適切に」ではなく「2スペースインデント」）
- **MUST**: ソースを明記する
- **SHOULD**: コード例は最小限（1つで十分）
- **SHOULD**: `paths` frontmatter でスコープを指定する
- **NEVER**: 他ルールと内容を重複させない（`@ファイル名` で参照）

## 重複発生時の対応

- **MUST**: どちらの責務か判定し、適切な方に集約する
- **MUST**: もう一方からは `@ファイル名` で参照のみにする
