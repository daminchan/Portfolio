# 実装ワークフロー

## 絶対厳守ルール

1. **勝手に実装しない** - 必ず確認を取る
2. **勝手にコミット・プッシュしない** - ユーザー承認必須
3. **段階的に進める** - 一気に実装しない

---

## 実装前（必須）

### 1. 理解確認
```
【理解確認】
□ タスクの内容を正確に理解したか？
□ 不明点はないか？（あれば質問）
□ 影響範囲を把握したか？
```
**→ 不明点があれば必ず質問してから次へ**

### 2. 実装宣言
実装前に以下の形式で宣言すること：
```
【実装宣言】
■ タスク: [具体的なタスク名]
■ 適用スキル: [component-creator / page-creator / etc]
■ 適用ルール: [code-style / workflow / etc]
■ 変更ファイル: [変更予定のファイル一覧]
■ 実装方針: [どのように実装するか]
```
**→ 「この方針で進めてよいですか？」と確認**

---

## 実装中（都度確認）

### チェックリスト（各ファイル変更後に確認）
```
【実装進捗】
✅ [完了したファイル/タスク]
🔄 [現在作業中]
⏳ [未着手]

【チェック】
□ 命名規則に従っているか
□ Props型を定義したか
□ 定数はconstants.tsから取得しているか
□ 50行を超えるコンポーネントは分割したか
```

---

## 実装後（必須）

### リファクタリング確認
```
【リファクタリング確認】
□ 不要なコードを削除したか
□ 重複コードはないか
□ 型定義は適切か
□ コンソールログは削除したか
```

### ビルド確認
```bash
npm run build
```
**→ エラーがあれば修正**

### コミット・プッシュ（最重要）
```
【コミット確認】
変更ファイル:
- [ファイル1]
- [ファイル2]

コミットメッセージ案:
[メッセージ]
```
**→ 必ず「コミット・プッシュしてよいですか？」と確認**
**→ ユーザーの承認後にのみ実行**
