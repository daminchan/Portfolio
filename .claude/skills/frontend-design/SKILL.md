# Frontend Design — 幻想的 x 現代的ポートフォリオ

独自性のある、本格的なフロントエンドUIを作成するスキル。
「AIっぽい」凡庸なデザインを避け、記憶に残るインターフェースを生成する。

## Design Thinking（コーディング前に必ず実行）

### 1. Purpose
- このインターフェースは何の問題を解決するか？
- 誰が見るか？（採用担当者、エンジニア、デザイナー）

### 2. Tone — 幻想的かつ現代的
- **方向性**: 幻想的（ethereal/mystical）+ 現代的（modern/clean）
- ダーク基調に光のアクセント、深みのある色彩、浮遊感のあるレイアウト
- 実装ごとにトーンのバリエーションを探る（毎回同じにしない）

### 3. Differentiation
- **「この画面で一番記憶に残るものは何か？」** を常に問う
- 1つの強いインパクト要素 > 多数の弱い装飾

## Aesthetics Guidelines

### Typography
- **MUST**: 独自性のあるフォントを選択する
- **NEVER**: Inter, Roboto, Arial, system fonts 等の汎用フォントをメインに使わない
- **SHOULD**: ディスプレイフォント（見出し用）+ ボディフォント（本文用）を組み合わせる
- **SHOULD**: 日本語フォントは Noto Sans JP / Noto Serif JP をベースに、英字は個性的なフォントを選ぶ

### Color & Theme
- **MUST**: CSS変数でカラーパレットを管理する
- **SHOULD**: ドミナントカラー + シャープなアクセントカラーの構成
- **SHOULD**: ダーク基調（幻想的なトーンに合致）
- **NEVER**: 紫グラデーション on 白背景（典型的なAIデザイン）
- **NEVER**: 均等に分配された無難なカラーパレット

### Motion（最重要）
- **MUST**: ページロード時のオーケストレーション — stagger による順次出現
- **MUST**: スクロール連動アニメーション（whileInView）
- **SHOULD**: ホバー/フォーカス時のマイクロインタラクション
- **SHOULD**: ページ遷移アニメーション（AnimatePresence）
- **SHOULD**: スムーズスクロール（Lenis）
- **SHOULD**: パララックス効果、浮遊感のある動き
- 詳細: @.claude/rules/animation.md

### Spatial Composition
- **SHOULD**: 非対称レイアウト、グリッドを崩す要素
- **SHOULD**: 大胆なネガティブスペース
- **SHOULD**: オーバーラップ、斜めの流れ
- **NEVER**: 均等な左右対称の退屈なレイアウト

### Backgrounds & Visual Details
- **SHOULD**: グラデーションメッシュ、ノイズテクスチャ、幾何学パターン
- **SHOULD**: レイヤード半透明、ドラマチックなシャドウ
- **SHOULD**: 雰囲気のある背景効果（単色背景を避ける）

| ライブラリ | 用途 |

| Lenis | スムーズスクロール |
| GSAP | 複雑なタイムライン制御（必要時） |
| Three.js / React Three Fiber | 3D要素（必要時） |
